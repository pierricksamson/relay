Référence API

Un seul endpoint à retenir

Relay expose un unique endpoint public. Authentifiez-vous avec la clé générée dans votre dashboard, envoyez un message, il arrive en DM Discord.

POST /api/send

Corps de la requête (JSON)

ChampTypeDescription
api_keystringClé API générée depuis le dashboard (préfixe pk_).
messagestringContenu du message, 2000 caractères max (limite Discord).

Réponses

CodeSignification
200Message envoyé.
400Requête invalide (champ manquant, message trop long).
401Clé API invalide ou révoquée.
429Limite de débit atteinte.
502Le bot n'a pas pu délivrer le DM (utilisateur non joignable).

Exemples

import requests

response = requests.post(
    "{{ base_url }}/api/send",
    json={
        "api_key": "pk_votre_cle",
        "message": "Le job de backup s'est terminé avec succès.",
    },
)
print(response.status_code, response.json())
curl -X POST {{ base_url }}/api/send \
  -H "Content-Type: application/json" \
  -d '{"api_key": "pk_votre_cle", "message": "Deploy terminé ✅"}'
await fetch("{{ base_url }}/api/send", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    api_key: "pk_votre_cle",
    message: "Deploy terminé ✅",
  }),
});

Bon à savoir

  • Une clé révoquée depuis le dashboard cesse de fonctionner immédiatement.
  • Gardez vos clés hors de vos dépôts publics : traitez-les comme un mot de passe.
